🔐 Authentication Flow

POST /auth/login→ bcrypt verify→ Create session token→ Set td_session cookie (30d)→ Redirect /dashboard
Cookies: httpOnly · 30-day expiry · Optional secure flag (production) · Google OAuth alternative
Password Reset: Email + Security Question → Validate answer hash → Reset password (no email sending needed)
Default Admin: admin@teledrive.app / admin123 (auto-created if DB is empty)

📄 Google Sheets Integration

Two methods: per-user OAuth2 (primary) or admin service account (fallback).

Per-User OAuth2 (Recommended)
  1. Admin configures OAuth Client ID & Secret in System Settings
  2. User clicks "Connect Google" on Dashboard
  3. Redirects to Google login → authorization
  4. Tokens stored in user_credentials table
  5. Auto-refresh when expired
Service Account (Fallback)
  1. Create Google Cloud project
  2. Enable Sheets API & create service account
  3. Download JSON key file
  4. Set GOOGLE_CREDS_JSON in .env (path) or GOOGLE_CREDS_BASE64
  5. Share target sheet with service account email

GOOGLE_CREDS_BASE64GitHub SecretBase64-encoded service account JSON (for CI)

Cron Jobs & GitHub Actions

Two scheduling mechanisms: GitHub Actions for cloud-based cron, and APScheduler for the web app's built-in scheduler.

🛠 GitHub Actions Workflow

File: .github/workflows/daily-scrape.yml

0 5 * * *
Daily at 05:00 UTC (08:00 UTC+3). Also supports manual trigger via workflow_dispatch.
Workflow Steps:
Checkout repo→ Setup Python 3.11→ pip install→ Decode TG_SESSION (base64)→ Run main.py

🕒 Built-in Web Scheduler (APScheduler)

Runs APScheduler in-process for scheduling scrapes without GitHub.

Cron Mode
Run scrapes at a specific time daily.
Config saved to scheduler_config.json
Interval Mode
Run scrapes every N hours.
Thread-safe with locks.
Per-Process Scheduling: Each process can have its own schedule (schedule_time, schedule_interval_hours) and daily limits to prevent Telegram bans.

Under the Hood

Internal mechanics: Telethon connection, duplicate detection, rate limiting, SocketIO real-time updates, and safety thresholds.

📡 Telegram MTProto Connection

Session Management
Telethon creates a .session SQLite file storing auth keys.
Session persists across runs. One phone = one session.
Backup numbers tracked in telegram_numbers table.
Message Extraction (20+ fields)
text, msg_id, date_utc, time_utc, views, forwards, replies_count
reactions (emoji + counts), reactions_sum, media_type
urls, url_count, hashtags, mentions, is_forward, fwd_from_name, post_link

🔄 Duplicate Detection

SQLite: post_link UNIQUE constraint — duplicates silently fail (INSERT OR IGNORE).
Sheets: Checks existing post_link column before appending. Only new messages added.
Process: scraped_messages tracks (channel_username, message_id) with UNIQUE constraint.

📡 Real-Time Updates (SocketIO)

Scraper starts→ emit('scrape_log')→ WebSocket to browser→ Append to live log→ Update progress bar
async_mode: threading · CORS: configurable · Used for scrape progress, status updates, errors

🗃 Google Sheets Internals

Structure: One tab per channel (auto-created if missing).
Headers: scrape_date, channel, username, msg_id, date_utc, time_utc, text, views, forwards, replies_count, reactions, reactions_sum, media_type, urls, url_count, hashtags, mentions, is_forward, fwd_from_name, post_link
Dual Auth: Service account (fallback) + per-user OAuth2 tokens.
Limits: Max 10M cells/sheet · 300 req/min API quota (free tier).

📊 Audit Logging

Every action logged to audit_log in scraper.db.
Tracked: logins, scrapes, profile changes, errors, settings, exports.
Fields: timestamp, action, details (JSON), status, user_id, user_email.
