Step 2: Telegram API Credentials

Required for scraping Telegram channels

Getting your API ID and Hash

  1. 1

    Visit Telegram Developer Portal

    Go to my.telegram.org and sign in with your phone number

  2. 2

    Go to API Development Tools

    Click "API Development Tools" in the menu

  3. 3

    Create a new application

    Fill in any app name and short name. You will receive an API ID (number) and API Hash (string)

Adding credentials in TeleDrive

  1. 1

    Open Settings tab

    In the dashboard sidebar, click Settings

  2. 2

    Find the Telegram API section

    Enter your API ID, API Hash, Phone Number (with country code, e.g. +966...), and Session Name

  3. 3

    Save and test the connection

    Click Save, then click Test Connection to verify. On first use, you may need to run python auth_session.py locally to create the Telegram session file

💡
Per-user credentialsEach user can store their own Telegram credentials in the Settings tab. Credentials are stored securely in the database and are never exposed to other users.
⚠️
Telegram Session FileYou need to create a session file by running python auth_session.py locally first. This authenticates your phone number with Telegram. The session file (scraper_session.session) must be present in the project root for scraping to work.

Step 3: Google Sheets Setup (Optional)

Push scraped data directly to Google Sheets

  1. 1

    Create a Google Cloud Project

    Go to console.cloud.google.com → create a new project

  2. 2

    Enable Google Sheets API

    APIs & Services → Enable APIs → search "Google Sheets API" → Enable

  3. 3

    Create a Service Account & download JSON key

    IAM & Admin → Service Accounts → Create → Download the JSON key file

  4. 4

    Create & share a Google Sheet

    Create a new Google Sheet, then share it (Editor role) with the service account email from the JSON file (looks like name@project.iam.gserviceaccount.com)

  5. 5

    Add credentials in TeleDrive

    In Settings → Google Sheets section, paste the path to the JSON key file and your Sheet ID (from the URL: docs.google.com/spreadsheets/d/SHEET_ID/edit)

Step 4: Admin Panel

Manage users, settings, and subscriptions

Access the admin panel at /admin (only visible to admin users).

SectionWhat You Can Do
OverviewView system stats: total users, active sessions, profiles, messages
UsersCreate/edit/delete users, change roles (admin/user), activate/deactivate accounts
System SettingsToggle registration, configure Google OAuth, change app branding
SubscriptionsEnable/disable subscription system, manage plans (Free, Pro, Enterprise)

Step 5: Managing Users

How users register and how you manage them

Registration Flow

  1. 1

    User registers at /register

    They provide name, email, and password (with confirmation). Registration can be disabled in Admin → System Settings.

  2. 2

    User logs in and adds their own credentials

    Each user goes to Settings and adds their own Telegram API keys and Google Sheets credentials

  3. 3

    Admin can manage from admin panel

    You can promote users to admin, deactivate accounts, or change their subscription plan from /admin → Users

Roles

RoleCapabilities
userAccess dashboard, manage own profiles/processes, own credentials, run scrapes
adminEverything above + admin panel, manage all users, system settings, subscriptions

How TeleDrive Works

Understanding the core workflow

Core Concepts

ConceptDescription
ProfileA scraping target: one Telegram channel + export settings (format, location, Google Sheets)
ProcessA scrape job within a profile: date range, rolling (last X hours), or one-time. Tracks progress, daily limits, resume point
SchedulerAutomatically runs due processes at set intervals. Processes can also be run manually
DedupMessages are never scraped twice per channel, even across multiple processes

Typical Admin Workflow

  1. 1

    Login & add your Telegram credentials

    Settings tab → Telegram section → Add API ID, Hash, Phone

  2. 2

    Create a Profile

    Profiles tab → + New Profile → enter channel username, export settings

  3. 3

    Create a Process inside the profile

    Expand the profile → + Add Process → choose type (date_range for historical, rolling for daily)

  4. 4

    Run the process

    Click Run → watch real-time progress with message counts, dates, and status updates

  5. 5

    Set up the scheduler (optional)

    Scheduler tab → enable → choose daily time or interval. Processes with scheduling enabled will run automatically

Step 6: Production Deployment (cPanel)

Deploy TeleDrive on shared hosting with cPanel

⚠️
RequirementsYour cPanel hosting must support Python applications (via Setup Python App or Phusion Passenger). Most modern cPanel hosts support this. Check with your hosting provider if unsure.

Option A: cPanel "Setup Python App" (Recommended)

  1. 1

    Upload project files

    Use cPanel File Manager or FTP to upload all project files to a directory, e.g. /home/username/teledrive/

  2. 2

    Open "Setup Python App" in cPanel

    Go to cPanel → Software → Setup Python App → Create Application

  3. 3

    Configure the Python app

    Set these values:

    Python version3.9 or higher
    Application rootteledrive
    Application URLyour-domain.com (or a subdomain)
    Application startup filepassenger_wsgi.py
    Application Entry pointapplication
  4. 4

    Create the passenger_wsgi.py file

    Create this file in your project root:

# passenger_wsgi.py import sys import os # Add project directory to path sys.path.insert(0, os.path.dirname(__file__)) # Set environment variables os.environ['FLASK_ENV'] = 'production' # Import the Flask app from web_app import app as application
  1. 5

    Install dependencies via cPanel terminal

    Open the cPanel Terminal (or SSH), activate the virtual environment, and install:

# Enter the virtual environment (path shown in Setup Python App page) source /home/username/virtualenv/teledrive/3.9/bin/activate # Install all dependencies pip install -r requirements.txt # Create the data directory mkdir -p data
  1. 6

    Update the SECRET_KEY

    In web_app.py, change the secret key to a random, unique string:

# Generate a secret key python -c "import secrets; print(secrets.token_hex(32))" # Then set it in web_app.py: app.config['SECRET_KEY'] = 'your-generated-secret-key-here'
  1. 7

    Restart the application

    Go back to Setup Python App in cPanel and click Restart. Your app should now be live at your domain!

  2. 8

    Upload the Telegram session file

    Upload your scraper_session.session file (generated locally with python auth_session.py) to the project root on the server

Option B: VPS / Dedicated Server (Advanced)

# SSH into your server ssh user@your-server.com # Clone or upload the project cd /var/www git clone your-repo-url teledrive cd teledrive # Create virtual environment python3 -m venv venv source venv/bin/activate pip install -r requirements.txt # Install gunicorn for production pip install gunicorn # Create data directory mkdir -p data # Run with gunicorn (production WSGI server) gunicorn -w 1 -b 0.0.0.0:5000 --worker-class eventlet web_app:app # Or use a systemd service (recommended for auto-restart) # Create /etc/systemd/system/teledrive.service
# /etc/systemd/system/teledrive.service [Unit] Description=TeleDrive After=network.target [Service] User=www-data WorkingDirectory=/var/www/teledrive ExecStart=/var/www/teledrive/venv/bin/gunicorn -w 1 -b 127.0.0.1:5000 --worker-class eventlet web_app:app Restart=always [Install] WantedBy=multi-user.target
# Enable and start the service sudo systemctl enable teledrive sudo systemctl start teledrive # Then configure Nginx as reverse proxy: # proxy_pass http://127.0.0.1:5000;

Security Checklist for Production

Essential steps before going live

  1. ✓

    Change the default admin password

    Login → Settings → change password from admin123 to something strong

  2. ✓

    Change the SECRET_KEY in web_app.py

    Replace the default key with a random 64-character hex string: python -c "import secrets; print(secrets.token_hex(32))"

  3. ✓

    Disable debug mode

    In web_app.py at the bottom, change debug=True to debug=False (or remove it; Passenger handles this)

  4. ✓

    Use HTTPS

    Enable SSL via cPanel → SSL/TLS or use Let's Encrypt (free). Session cookies require HTTPS for security

  5. ✓

    Protect the data/ directory

    Ensure data/teledrive.db is not web-accessible. Add to .htaccess:

# .htaccess — block access to sensitive files <FilesMatch "\.(db|session|env|py|pyc)$"> Deny from all </FilesMatch> <DirectoryMatch "^/.*/data/"> Deny from all </DirectoryMatch>
  1. ✓

    Control registration

    If you don't want public signups, disable registration in Admin → System Settings. Create user accounts manually from the admin panel

  2. ✓

    Backup the database regularly

    The SQLite database is at data/teledrive.db. Set up a cron job to back it up: cp data/teledrive.db data/backup_$(date +%Y%m%d).db

✅
You're production-ready!After completing all steps above, your TeleDrive instance is secure and ready for real-world use.
